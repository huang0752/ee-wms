# WMS V1 简化决策与待确认问题

本文记录 WMS V1 的产品决策。用户已确认第一版按简单方案实现，因此以下“V1 决策”作为后续开发默认依据；复杂能力只做字段、枚举、接口边界预留。

## 1. V1 已确认简化决策

| 编号 | 问题 | V1 决策 | 扩展预留 |
|---|---|---|
| D-01 | SN 单件管理 | 不做完整 SN 作业闭环。库存以批次为主，SN 字段可为空。 | 物料保留 `sn_enabled`，库存批次/流水保留 `sn_code`，后续可启用 SN 库存页面。 |
| D-02 | PDA 形态 | 不做原生 App。V1 做 H5/响应式扫码作业页，兼容电脑扫码枪输入。 | 路由和接口按移动端作业场景拆分，后续可封装 UniApp/App。 |
| D-03 | IQC 来料检验 | 不等待 MES。V1 在 WMS 内做轻量 IQC 任务、判定和状态控制。 | 保留 `external_quality_id`、`mes_quality_status` 等外部质量系统字段。 |
| D-04 | ERP/MES 集成 | 不做实时自动集成。V1 支持手工录入、Excel 导入和接口契约预留。 | 所有业务单据保留 `external_source`、`external_id`、`external_no`、`sync_status`。 |
| D-05 | 审批流 | 不接工作流引擎。V1 用状态机、按钮权限和操作日志完成审核。 | 关键单据保留 `audit_status`、`audited_by`、`audited_time`、`workflow_instance_id`。 |

## 2. 其他默认决策

| 编号 | 问题 | V1 默认决策 |
|---|---|---|
| Q-01 | 仓库类型 | 固定预置原材料仓、半成品仓、成品仓、辅料仓、不良品仓，允许字典扩展。 |
| Q-02 | 条码规则 | 使用租户级默认规则：物料码、批次码、库位码、单据码；可配置租户短码、对象前缀、日期格式和流水位数，不做复杂脚本引擎。 |
| Q-03 | 设备接口 | 电子秤、RFID、立库、AGV 只预留接口和枚举，不接硬件协议。 |
| Q-04 | 试用数据 | 允许重复生成；同一租户同一时间只允许一个生成任务运行；清理只影响演示批次。 |
| Q-05 | 销售出库 | V1 支持基础销售出库，但优先级低于采购入库、生产领料、实时库存。 |
| Q-06 | 财务库存 | V1 不做成本核算和财务账，只保留金额字段用于展示和后续 ERP 对接。 |
| Q-07 | 多工厂 | 使用租户隔离 + 部门树表达组织，不新增工厂平台级模型。 |
| Q-08 | 离线作业 | 不做离线缓存；网络异常给明确提示。 |

## 3. 后续仍需业务确认

这些问题不阻塞第一阶段开发，可在入库/出库阶段前补充：

1. 客户现场是否有固定物料编码、批次号、库位号规则。
2. 生产领料是否必须按工单 BOM 自动生成，还是允许人工创建领料单。
3. 来料检验是否需要抽样数量、缺陷项、检验附件。
4. 销售出库是否需要客户签收、物流单号、发货附件。
5. 盘点是否需要多人复盘、盲盘、复盘差异确认。
