# WMS 待确认问题

本文列出需要你确认的问题。为保证开发能推进，每个问题都给出默认假设；如果你没有特别说明，后续开发按默认假设执行。

## 1. 业务范围确认

| 编号 | 问题 | 默认假设 |
|---|---|---|
| Q-01 | 仓库类型是否固定包含原材料仓、半成品仓、成品仓、辅料仓、不良品仓？ | V1 固定预置这 5 类，允许客户在字典中扩展。 |
| Q-02 | 是否需要 SN 单件管理？ | V1 数据模型预留 SN 字段和 SN 库存页面入口，但首个开发切片以批次管理为主。 |
| Q-03 | 企业是否已有条码规则标准？ | V1 使用系统默认规则：物料码、批次码、库位码、单据码；规则可配置但不做复杂脚本引擎。 |
| Q-04 | PDA 是安卓 App、浏览器 H5，还是电脑端扫码枪？ | V1 做浏览器 H5/PWA 风格移动页面，同时兼容电脑端扫码枪输入。 |
| Q-05 | 来料检验由 WMS 自己做，还是 MES 质量模块做？ | V1 在 WMS 内建立轻量 IQC 任务和判定结果；后续可同步 MES 质量模块。 |
| Q-06 | 是否已有 ERP，需要同步采购订单、销售订单、物料、BOM？ | V1 先支持手工录入和 Excel 导入；接口层预留 ERP/MES 外部单号。 |
| Q-07 | 是否需要电子秤、RFID、立库、AGV？ | V1 只预留设备接口表和枚举，不接硬件协议。 |
| Q-08 | 试用数据是否允许租户重复生成？ | 允许重复生成，但同一租户同一时间只允许一个生成任务运行，清理只影响演示批次。 |

## 2. 产品策略确认

| 编号 | 问题 | 默认假设 |
|---|---|---|
| Q-09 | WMS 和 MES 后续是否都作为独立产品仓库维护？ | 是。共享能力进入框架，业务对象留在各自产品仓库。 |
| Q-10 | WMS 首版是否必须做完整销售出库？ | V1 支持销售出库基础流程，但优先级低于采购入库、生产领料和实时库存。 |
| Q-11 | 是否需要金额、库存成本、财务账？ | V1 不做财务成本核算，只保留金额字段用于驾驶舱展示和后续 ERP 对接。 |
| Q-12 | 是否需要多工厂、多公司？ | 使用租户隔离 + 部门树表达组织；V1 不新增“工厂”平台级概念。 |
| Q-13 | 是否要求移动端离线作业？ | V1 不做离线缓存；扫码失败、网络异常给明确提示。 |
| Q-14 | 是否需要审批流？ | V1 盘点调整、库存冻结等先用状态和权限控制；流程引擎留到 P1/P2。 |

## 3. 请优先回复的问题

请优先确认这 5 个问题，它们会影响第一阶段表结构和页面：

1. `Q-02`：SN 单件管理是 V1 必做，还是只预留？
2. `Q-04`：PDA 首版按 H5 做是否可以？
3. `Q-05`：IQC 首版放 WMS 内部是否可以？
4. `Q-06`：ERP/MES 首版是否只做接口预留和手工/导入数据？
5. `Q-14`：盘点调整首版用权限审核，不接工作流是否可以？
