# WMS电工装备智慧仓储系统 BRD + PRD

## 1. 文档信息

| 项目 | 内容 |
|---|---|
| 系统名称 | WMS电工装备智慧仓储系统 |
| 文档类型 | BRD + PRD |
| 适用场景 | 电工装备制造企业原材料、半成品、成品仓储管理与批次追溯 |
| 不纳入范围 | AGV 调度算法、自动立库底层控制、运输 TMS、财务库存核算 |
| 关联系统 | MES电工装备智慧物联系统、ERP/采购系统、扫码/RFID/PDA 设备 |
| 目标版本 | V1.0 |

### 1.1 系统边界

WMS 应作为独立业务系统建设，不作为当前 FastapiAdmin 主项目中的普通插件交付。

| 边界项 | 要求 |
|---|---|
| 前端 | WMS 独立前端工程或独立部署包，复用 Art Design Pro / `fa-*` 组件体系和页面规范 |
| 后端 | WMS 独立 FastAPI 后端，复用 FastapiAdmin 的认证、权限、租户、日志、上传、任务、AI 调用等框架能力 |
| 数据库 | WMS 独立数据库，仓储、库存、批次、出入库、盘点等业务表不与 MES 共库 |
| 租户 | WMS 内维护自己的租户、角色、部门、菜单和业务数据隔离 |
| 插件机制 | 可复用框架 `module_*` 命名、动态路由、套餐授权等思想，但 WMS 本身不是一个可随意安装/卸载的轻量插件 |
| 与 MES | 通过 API、消息或数据同步对接，不通过共享数据库强耦合 |

因此，文档中的 `module_wms:*` 权限和 `module_wms` 命名是 WMS 独立系统内部的模块命名约定，不表示把整个 WMS 做成 FastapiAdmin 主项目内的一个普通插件。

## 2. BRD 商业需求

### 2.1 背景

电工装备制造企业通常存在物料种类多、批次多、规格型号复杂、生产领料频繁、质量追溯要求高等特点。传统仓储依赖人工台账和线下交接，容易出现账实不一致、批次混用、缺料停工、呆滞积压、追溯困难等问题。

WMS电工装备智慧仓储系统用于建立原材料、半成品、成品全流程仓储数字化管理能力，通过条码/RFID、PDA 扫码、库区库位、批次/SN 管理、安全库存预警和 MES 联动，实现仓储作业无纸化、库存状态实时化、物料流转可追溯。

### 2.2 建设目标

1. 建立仓库、库区、库位、物料、批次统一管理体系。
2. 支持采购到货、来料检验、原材料入库、生产领料、生产退料、半成品入库、成品入库、销售出库等仓储流程。
3. 支持条码/RFID 作业，减少人工录入和错发、漏发。
4. 支持批次、SN、供应商、检验状态、库存状态的全生命周期追踪。
5. 与 MES 工单、BOM、工序完工、成品入库联动，支撑生产全过程追溯。
6. 建立安全库存、呆滞库存、效期、缺料等预警机制。

### 2.3 核心价值

| 价值方向 | 说明 |
|---|---|
| 评审支撑 | 直接支撑“智能仓储”“原材料检测”“物联采集”“自动监控”“批次追溯”等表述 |
| 库存准确 | 入出库扫码、库位管理、盘点调整提升账实一致 |
| 生产协同 | 按 MES 工单/BOM 自动生成备料、领料任务 |
| 质量管控 | 来料检验、冻结、退货、不合格品隔离 |
| 追溯闭环 | 原材料批次从到货、检验、入库、领料到成品可追踪 |

### 2.4 用户角色

本系统不单独实现一套用户、部门、角色体系，继承 FastapiAdmin 已有系统管理能力：

| 框架能力 | 继承方式 |
|---|---|
| 用户管理 | 使用 `sys_user`，用户可归属部门、岗位、角色 |
| 部门管理 | 使用 `sys_dept`，支持树形部门，如仓储部、原料仓、成品仓、质检组 |
| 角色管理 | 使用 `sys_role`，业务角色可由实施人员自定义 |
| 菜单权限 | 使用 `platform_menu` + `sys_role_menus`，控制 WMS 菜单和按钮 |
| 数据权限 | 使用角色 `data_scope`：仅本人、本部门、本部门及以下、全部、自定义部门 |
| 多租户 | 使用现有租户模型，支持后续多公司/多仓库隔离 |

下表不是写死的系统角色，而是 WMS 首版建议预置角色。客户可在系统管理中增删改角色、调整菜单权限和数据范围。

| 建议业务角色 | 典型部门 | 关注点 | 主要操作 |
|---|---|---|---|
| 仓储主管 | 仓储部 | 库存准确、作业效率、预警风险 | 查看仓储驾驶舱、审批调整、监督作业 |
| 原材料仓管员 | 原材料仓 | 到货、入库、上架、发料 | 收货、入库、生产领料、盘点 |
| 成品仓管员 | 成品仓 | 成品入库、出库、库存准确 | 成品入库、销售出库、盘点 |
| 不良品仓管员 | 不良品仓 | 不合格品隔离与流转 | 不良入库、冻结、退货、处置 |
| 质检员 | 质量部/来料检验组 | 来料检验、库存状态判定 | IQC 检验、合格/不合格/冻结判定 |
| 生产领料员 | 车间/班组 | 工单领料、退料、补料 | 发起或确认领料、退料 |
| 采购员 | 采购部 | 到货状态、检验结果、退货 | 查看到货、检验、入库情况 |
| 生产计划员 | 计划部 | 物料齐套、缺料风险 | 查询工单齐套、库存可用量 |
| 系统管理员 | 信息部 | 基础资料、权限、条码规则 | 配置仓库、角色、部门、参数、条码模板 |

### 2.6 组织与权限配置原则

建议在现有部门树下补齐仓储组织，不在 WMS 业务表里重复维护组织架构。

```text
公司
  - 仓储部
    - 原材料仓
    - 半成品仓
    - 成品仓
    - 辅料仓
    - 不良品仓
  - 质量部
    - 来料检验组
  - 生产管理中心
    - 计划部
    - 一车间
    - 二车间
  - 采购部
  - 信息部
```

业务数据建议统一记录 `tenant_id`、`dept_id`、`created_id`、`updated_id` 等框架通用字段，使角色数据权限可以直接生效。仓库、库区、库位、库存流水、入库单、出库单、盘点单等对象还应记录业务责任部门，例如仓库归属部门、作业部门、检验部门、领料部门。

### 2.7 成功指标

| 指标 | 目标 |
|---|---|
| 入出库扫码覆盖率 | 关键物料 100% 支持条码/RFID 作业 |
| 批次追溯覆盖率 | 原材料、半成品、成品批次可追溯 |
| 库存准确率 | 账实一致率持续提升，目标 98%+ |
| 缺料预警 | 工单下达后可提示关键物料缺口 |
| 来料检验闭环 | 到货、检验、入库、退货状态可追踪 |

1. 仓储驾驶舱

3. 物料批次与 SN 管理
4. 条码/RFID 规则和打印
5. 采购到货与收货
6. 来料检验联动
7. 原材料入库、上架
8. 生产领料、补料、退料
9. 半成品、成品入库

11. 移库、调拨、盘点、库存调整

### 3.2 可继承的框架能力

WMS 系统应尽量复用 FastapiAdmin 现有后台框架能力，只新增电工装备仓储业务对象、业务接口和业务页面。本项目 Web 前端基于 Art Design Pro 风格体系演进，当前代码中大量能力已沉淀为 `fa-*` 组件、全局样式、布局壳和 dashboard 模板；WMS 页面应优先复用这些能力，而不是重新编写通用表格、图表、弹窗、卡片、布局、主题和权限组件。

| 能力类别 | 本项目已有能力 | WMS 使用方式 |
|---|---|---|
| Art Design Pro 基础 | 项目保留 Art Design Pro 设计体系、交互习惯和组件封装，当前命名多为 `fa-*` | WMS 页面遵循现有后台风格，优先找现成组件再补业务适配 |
| 应用布局 | `frontend/web/src/components/layouts`，包含侧栏、顶部栏、面包屑、工作台标签、全局搜索 | WMS 菜单直接接入现有布局，不单独做后台壳 |
| 菜单路由 | `platform_menu`、动态路由、`MenuProcessor`、静态/后端混合路由 | WMS 独立系统内使用 `module_wms` 菜单树命名，支持目录、页面、按钮权限 |
| 主题与视觉 | Element Plus、Tailwind、全局 `fa-card`、暗色/浅色主题、设置面板 | WMS 页面沿用现有视觉系统，避免另起一套 UI |
| 仪表盘模板 | `frontend/web/src/views/dashboard/workplace`、`analysis`、`screen` | 仓储驾驶舱复用工作台/分析页/大屏页布局和样式 |
| 图表组件 | `components/ECharts`、`components/charts/fa-line-chart`、`fa-bar-chart`、`fa-ring-chart`、`fa-radar-chart`、`fa-map-chart` 等 | 库存趋势、库龄分析、出入库趋势、预警分布直接复用图表组件 |
| 指标卡片 | `fa-stats-card`、`fa-line-chart-card`、`fa-bar-chart-card`、`fa-donut-chart-card`、`fa-progress-card` | 库存 KPI 卡、预警卡、周转卡直接复用 |
| 表格与查询 | `fa-table`、`fa-search-bar`、`fa-pagination`、导入导出组件 | 到货、入库、出库、库存、盘点等列表页复用通用表格体验 |
| 弹窗抽屉 | `fa-dialog`、`fa-drawer`、`fa-export-dialog`、`fa-import-dialog` | 入库确认、出库复核、库存详情、盘点差异复用 |
| 文本/数字动效 | `fa-count-to`、`fa-text-scroll` | 仓储 KPI 数字滚动、预警滚动消息复用 |
| WebSocket 客户端 | `frontend/web/src/utils/socket`，支持连接管理、心跳、重连、消息队列 | 后续扫码作业结果、库存预警推送可复用，V1.0 可先轮询 |
| 通知与角标 | `fa-notification`、菜单 `show_badge/show_text_badge` | 待收货、待检、待发料、库存预警可显示通知和菜单角标 |
| 系统监控 | 后端已有服务器监控、缓存监控、在线用户接口 | WMS 运维页可复用系统监控，不纳入 WMS 业务开发 |
| 权限体系 | 用户、角色、部门、菜单、按钮、数据权限 | WMS 只定义权限点，执行交给框架 |
| 操作日志 | `OperationLogRoute` 和系统日志模块 | 入库、出库、移库、冻结、盘点调整等关键操作记录日志 |
| 文件上传 | 通用上传、附件能力 | 到货附件、质检附件、盘点附件、条码模板复用 |
| 代码生成 | 代码生成器模板 | 基础 CRUD 可由生成器生成后再补业务逻辑 |

#### 3.2.1 插件边界说明

FastapiAdmin 当前已有插件机制，适合承载 AI、任务、代码生成、工作流、数据大屏等框架可选能力。WMS 不建议作为该插件市场中的普通插件交付，原因如下：

1. WMS 是完整仓储业务系统，包含库存账、库存流水、批次状态、出入库流程和设备接口。
2. WMS 需要独立数据库和独立部署生命周期，不适合随租户在主项目中安装/卸载。
3. WMS 可复用插件机制中的动态模块、菜单权限、套餐授权思想，但交付边界应是独立系统。
4. WMS 内部可继续使用 `module_wms` 包名、权限前缀和菜单命名，方便沿用框架生成器和权限规范。

#### 3.2.2 组件复用原则

1. 业务页面优先复用 `frontend/web/src/components` 下的 `fa-*` 组件。
2. 列表页优先使用 `fa-table`、`fa-search-bar`、`fa-pagination`、`fa-button-table`。
3. 表单页优先使用 `fa-form`、Element Plus 表单控件和现有校验风格。
4. 详情页优先使用 `fa-descriptions`、`fa-drawer`、`fa-dialog`。
5. 导入导出优先使用 `fa-excel-import`、`fa-excel-export`、`fa-import-dialog`、`fa-export-dialog`。
6. 状态展示优先使用 `fa-status-tag`、全局 badge 样式、Element Plus Tag。
7. 图表和看板优先使用 `components/charts`、`components/cards`、`views/dashboard` 现有模板。
8. 图标优先使用现有 `FaSvgIcon` + Iconify/Remix 图标，不手写 SVG。
9. 业务模块不得复制粘贴一套新的基础组件；确实缺少通用能力时，应沉淀到 `components/fa-*`，再由 MES/WMS 共用。
10. 页面样式优先使用全局 `fa-card`、`fa-table-card`、`fa-full-height`、主题变量和 Tailwind 工具类，避免在业务页面写大量孤立样式。

#### 3.2.3 业务页面复用映射

| WMS 页面类型 | 优先复用 |
|---|---|
| 到货/入库/出库列表 | `fa-table` + `fa-search-bar` + `fa-button-table` + `fa-status-tag` |
| 单据详情 | `fa-drawer` + `fa-descriptions` + Element Plus Tabs |
| 仓库/库区/库位维护 | `fa-table` + `fa-form` + `fa-dialog` |
| 扫码作业页面 | `fa-form` + `fa-icon-button` + `fa-status-tag` |
| 实时库存查询 | `fa-table` + `fa-search-bar` + `fa-excel-export` |
| 库存盘点 | `fa-table` + `fa-excel-import/export` + `fa-drawer` |
| 批次追溯 | `fa-descriptions` + `fa-timeline-list-card` + `fa-table` |
| 仓储驾驶舱 | `dashboard/analysis`、`dashboard/screen`、`fa-stats-card`、`fa-line-chart`、`fa-ring-chart` |

### 3.3 仪表盘继承策略

WMS 不新造一套“高级看板框架”，而是基于现有 Dashboard 能力扩展业务页面：

| 页面形态 | 继承来源 | WMS 页面 |
|---|---|---|
| 常规工作台 | `views/dashboard/workplace` | WMS 工作台：待收货、待检、待发料、待盘点、快捷入口 |
| 分析型仪表盘 | `views/dashboard/analysis` | 仓储分析：库存结构、出入库趋势、库龄、周转、预警 |
| 全屏大屏 | `views/dashboard/screen` | 仓储大屏：实时库存、任务状态、预警滚动、库存趋势 |
| 图表卡片 | `components/cards/*chart-card` | 库存趋势卡、出入库趋势卡、预警卡 |
| 通用图表 | `components/charts/*` | 折线、柱状、环形、排行、雷达图 |

业务开发只需要新增：

1. WMS 统计接口，例如 `/wms/dashboard/summary`、`/wms/dashboard/trends`、`/wms/dashboard/warnings`。
2. WMS 图表数据适配层，把业务指标转换为现有图表组件需要的数据结构。
3. WMS 大屏页面配置，将现有 `dashboard/screen` 的交易类文案替换为仓储类指标。

### 3.4 不包含范围

1. AGV 路径规划和调度算法。
2. 自动立库 PLC 控制。
3. 运输配送 TMS。
4. 财务成本核算和总账库存。
5. 供应商门户，V1.0 可先由内部人员录入到货。

### 4.2 仓储驾驶舱

#### 4.2.1 功能目标

展示仓储运营状态、库存风险、作业任务和 MES 生产协同情况。

仓储驾驶舱应继承本项目现有 dashboard 能力：

1. 常规页面复用 `dashboard/analysis` 的响应式卡片布局。
2. 大屏页面复用 `dashboard/screen` 的全屏容器、顶部指标卡、图表网格、底部滚动状态栏。
3. 指标卡复用 `fa-stats-card` 和 `fa-count-to`。
4. 趋势图复用 `fa-line-chart`、`fa-bar-chart`、`fa-ring-chart`。
5. 预警消息可复用文本滚动和通知组件。
6. 数据刷新 V1.0 可用接口轮询，后续接入现有 WebSocket 客户端。

#### 4.2.2 指标

| 指标 | 说明 |
|---|---|
| 库存总量 | 当前库存数量/金额，金额可 V2.0 接 ERP |
| 可用库存 | 扣除冻结、待检、锁定后的库存 |
| 待收货任务 | 未完成采购到货数量 |
| 待检验任务 | IQC 未完成数量 |
| 待发料任务 | MES 工单领料待处理数量 |
| 成品待入库 | MES 完工后待入库数量 |
| 安全库存预警 | 低于阈值物料数量 |
| 缺料预警 | 影响生产工单的缺料项 |
| 呆滞库存 | 超过指定天数无出库物料 |

#### 4.2.3 推荐大屏布局

```text
WMS仓储大屏
```

### 4.3 仓储基础

#### 4.3.1 仓库档案

| 字段 | 说明 |
|---|---|
| 仓库编码 | 唯一编码 |
| 仓库名称 | 原材料仓、半成品仓、成品仓、不良品仓等 |
| 仓库类型 | 原料、半成品、成品、辅料、不良品 |
| 负责人 | 仓库负责人 |
| 状态 | 启用/停用 |

#### 4.3.2 库区库位

1. 支持仓库 -> 库区 -> 库位三级结构。
2. 库位可配置容量、承重、适用品类、是否混放。
3. 支持固定库位、随机库位、ABC 分类存储策略。

#### 4.3.3 条码规则

| 对象 | 条码内容 |
|---|---|
| 物料 | 物料编码 |
| 批次 | 物料编码 + 批次号 |
| SN | 物料编码 + 序列号 |
| 库位 | 仓库 + 库区 + 库位 |
| 单据 | 入库单、出库单、领料单、盘点单 |

### 4.4 到货与检验

#### 4.4.1 采购到货

| 字段 | 说明 |
|---|---|
| 到货单号 | 系统生成或 ERP 同步 |
| 供应商 | 关联供应商档案 |
| 物料 | 关联物料档案 |
| 到货数量 | 实际到货数量 |
| 采购批次 | 供应商批次或系统批次 |
| 到货日期 | 到货时间 |
| 状态 | 待收货、已收货、待检、已入库、退货 |

#### 4.4.2 来料检验联动

1. 收货后可自动生成 IQC 来料检验任务。
2. 检验合格后允许入库。
3. 检验不合格可退货、让步接收、冻结、转不良品仓。
4. 检验记录同步给 MES 质量模块，用于原材料质量追溯。

### 4.5 入库管理

#### 4.5.1 原材料入库

1. 选择合格到货批次生成入库单。
2. 扫描物料条码、批次条码、库位条码完成上架。
3. 系统按策略推荐库位。
4. 入库后更新批次库存和出入库流水。

#### 4.5.2 半成品/成品入库

1. MES 工序完工或工单完工后生成入库任务。
2. 扫描工单、产品批次、库位完成入库。
3. 成品入库前可校验 OQC/出厂试验结果。
4. 入库后支持成品批次追溯。

### 4.6 出库管理

#### 4.6.1 生产领料

1. MES 工单下达后，WMS 按 BOM 生成备料/领料任务。
2. 系统根据 FIFO、批次状态、库存可用量推荐出库批次。
3. 仓管员扫码拣货，生产领料员确认。
4. 领料完成后把物料批次回传 MES。

#### 4.6.2 生产补料与退料

| 功能 | 说明 |
|---|---|
| 补料 | 工单物料不足或损耗超标时发起补料 |
| 退料 | 工单剩余物料退回仓库 |
| 批次保持 | 退料需保留原领料批次 |
| 状态判定 | 可用、待检、冻结、不良 |

#### 4.6.3 销售出库

1. 根据销售出库单或人工出库单生成拣货任务。
2. 支持扫码拣货、复核、出库确认。
3. 出库后形成客户、成品批次、出库时间追溯关系。

### 4.7 库存管理

#### 4.7.1 实时库存

| 字段 | 说明 |
|---|---|
| 物料 | 物料编码、名称、规格 |
| 仓库/库位 | 当前所在位置 |
| 批次/SN | 批次号或序列号 |
| 库存数量 | 账面数量 |
| 可用数量 | 可发料数量 |
| 锁定数量 | 已分配未出库数量 |
| 待检数量 | 收货后未检验数量 |
| 冻结数量 | 不可用库存 |

#### 4.7.2 库存状态

| 状态 | 说明 |
|---|---|
| 待检 | 到货后未完成检验 |
| 可用 | 检验合格，可领料/出库 |
| 锁定 | 已被工单或出库单占用 |
| 冻结 | 质量异常或人工冻结 |
| 不良 | 不合格品库存 |
| 已出库 | 完成出库 |

#### 4.7.3 移库调拨

1. 支持库位间移库。
2. 支持仓库间调拨。
3. 全程扫码确认。
4. 记录操作人、时间、原库位、目标库位。

#### 4.7.4 库存盘点

1. 支持按仓库、库区、物料分类创建盘点单。
2. 支持明盘、盲盘。
3. 支持扫码盘点。
4. 生成盘盈、盘亏差异。
5. 审批后形成库存调整。

### 4.8 预警与分析

| 预警 | 规则 |
|---|---|
| 安全库存预警 | 可用库存低于安全库存 |
| 缺料预警 | MES 工单需求量大于可用库存 |
| 呆滞库存 | 指定天数无出库 |
| 效期预警 | 临近失效日期 |
| 超储预警 | 库存高于最大库存 |
| 待检超时 | 到货后超过指定时间未检验 |

### 4.9 条码/RFID/PDA 作业

#### 4.9.1 扫码场景

1. 到货收货扫码。
2. 原材料入库扫码。
3. 库位上架扫码。
4. 生产领料扫码。
5. 成品入库扫码。
6. 销售出库扫码。
7. 移库调拨扫码。
8. 库存盘点扫码。

### 4.10 仓储追溯

#### 4.10.1 批次追溯

输入物料批次或成品批次后展示：

1. 供应商。
2. 到货单。
3. 检验记录。
4. 入库记录。
5. 库位变更。
6. 领料工单。
7. 生产工单。
8. 成品批次。
9. 出库客户。

#### 4.10.2 追溯方向

1. 正向：原材料批次 -> 生产工单 -> 成品批次 -> 出库客户。
2. 反向：成品批次 -> 生产工单 -> 原材料批次 -> 供应商。

### 4.13 智能化能力规划

WMS 的“智能”不应只表现为条码/RFID 或页面大屏，而应体现在仓储作业自动判断、自动推荐、自动预警、自动归因和自动总结。V1.0 应优先实现规则可控、结果可解释的智能能力；AI 主要增强说明、归因、建议和试用数据真实感。

#### 4.13.1 智能能力分层

| 层级 | 能力 | V1.0 定位 |
|---|---|---|
| 自动感知 | 入库、出库、库存、库位、批次状态自动汇总 | 必须 |
| 自动预警 | 安全库存、缺料、超储、呆滞、效期、冻结库存预警 | 必须 |
| 自动推荐 | 推荐库位、推荐出库批次、推荐补货、推荐盘点对象 | 必须 |
| 自动分析 | 出入库趋势、库龄分析、周转分析、库存结构分析 | 必须 |
| AI 增强 | 库存摘要、差异原因说明、预警处理建议、试用数据生成 | 建议 |
| 预测优化 | 需求预测、动态补货、库位优化、自动调拨建议 | 后续 |

#### 4.13.2 V1.0 必须体现的智能点

| 场景 | 智能表现 | 说明 |
|---|---|---|
| 采购入库 | 自动推荐上架库位 | 根据物料分类、库位容量、库区属性、批次状态推荐 |
| 生产领料 | 自动推荐出库批次 | 按 FIFO、批次状态、冻结状态、可用库存推荐 |
| 库存管理 | 自动生成库存预警 | 安全库存、超储、呆滞、效期、冻结库存自动提醒 |
| 缺料分析 | 自动识别缺料风险 | 根据生产需求和可用库存计算缺口 |
| 盘点管理 | 自动推荐盘点范围 | 根据高价值、高频出入库、长期未盘点、差异频发物料推荐 |
| 批次追溯 | 自动串联批次流向 | 输入批次后展示入库、检验、移库、领料、出库链路 |
| 仓储驾驶舱 | 自动生成库存摘要 | 用自然语言概括库存风险、待办、周转、异常 |
| 作业异常 | 自动提示处理路径 | 库存不足、批次冻结、库位容量不足时给出下一步建议 |

#### 4.13.3 AI 在 WMS 中的具体用法

| AI 功能 | 使用对象 | 输出 |
|---|---|---|
| 仓储日报摘要 | 仓储驾驶舱 | 今日入库、出库、库存预警、待处理事项摘要 |
| 预警处理建议 | 安全库存、呆滞、效期、缺料预警 | 建议补货、调拨、冻结、优先发料等处理意见 |
| 盘点差异说明 | 盘点单 | 可能原因、复核建议、调整说明草稿 |
| 库存结构解读 | 库存分析 | 高库存、低周转、风险物料说明 |
| 试用数据增强 | 租户试用数据初始化 | 仓库命名、物料规格、供应商、异常说明、场景文案 |

#### 4.13.4 智能能力边界

1. 库存数量、可用量、冻结量、批次状态必须由规则和库存流水计算，不能由 AI 生成结论。
2. AI 建议只能作为辅助说明，不能自动执行出入库、调整、冻结、解冻。
3. 库存调整、盘盈盘亏、批次冻结等关键动作必须由授权用户确认。
4. AI 调用统一走平台默认模型配置，WMS 不单独维护模型密钥。
5. AI 不可用时，预警、推荐、驾驶舱和核心仓储作业仍需正常运行。

### 4.14 租户试用数据初始化

#### 4.14.1 业务定位

WMS 系统需要提供“租户试用数据初始化”能力，用于客户租户管理员首次进入系统后，自助生成一套可操作、可查看、可重置的仓储试用数据。

该能力不是平台管理员手工导入固定样例，也不是单纯数据库 seed。它应根据租户填写的企业画像，生成更贴近电工装备仓储现场的数据，帮助客户体验仓库库位、批次库存、入库、出库、盘点、条码、库存预警、追溯和驾驶舱。

#### 4.14.2 与平台 AI 能力的关系

AI 模型配置属于平台能力：

- 只能平台管理员维护 AI 模型配置。
- 平台支持多个模型配置，但同一时间只有一个默认启用模型。
- 租户管理员和普通用户不能配置模型、Base URL、API Key。
- WMS 只调用平台统一 AI 服务，不在 WMS 内单独维护模型配置。
- AI 不可用时，WMS 仍应使用内置规则模板生成基础试用数据。

WMS 中 AI 的使用边界：

| 内容 | 生成方式 |
|---|---|
| 主外键、数量、库存流水、状态、时间线 | 规则引擎生成 |
| 仓库/库区/库位命名 | AI + 行业词库 |
| 物料名称、规格型号、供应商名称 | AI 增强 |
| 盘点差异原因、库存预警说明 | AI 增强 |
| 驾驶舱摘要、试用场景说明 | AI 增强 |

#### 4.14.2.1 电工装备样本池边界

WMS 内置一份“电工装备默认样本池”，用于演示数据生成的提示词、行业词库和规则模板。样本池可以参考外部物资范围资料，但不作为正式业务种子、不写入正式物料目录、不形成供应商准入目录。

租户管理员可以复制系统默认样本池并维护自己的样本池副本，用于调整演示数据的产品方向、物料命名、供应商命名、规格模板、异常说明和生成提示词。租户只能修改本租户样本池，不能修改系统内置样本池，也不能影响其他租户。

样本池只服务 `is_demo = true` 的试用数据生成。正式物料、供应商、客户、库存和单据仍由租户业务流程维护，不反向依赖样本池。

#### 4.14.3 租户填写内容

| 字段 | 说明 |
|---|---|
| 企业名称 | 默认可取租户名称，允许租户管理员调整展示名 |
| 企业规模 | 小型、中型、大型 |
| 仓储场景 | 原材料仓、半成品仓、成品仓、辅料仓、不良品仓 |
| 产品方向 | 开关柜、箱式变电站、变压器、断路器、电缆附件、综合电气成套 |
| 自有产品 | 租户可填写自己的产品名称、品类、规格示例、电压等级、仓储特性、质量要求、供应商要求 |
| 仓库规模 | 仓库数量、库区数量、库位数量 |
| 数量目标 | 物料数量、库存流水数量、业务单据数量、预警数量，可选择固定模式或自定义 |
| 条码模式 | 批次条码、物料条码、库位条码 |
| 试用时间范围 | 最近 30 天、90 天、180 天 |
| 数据规模 | 标准、丰富、压力测试 |
| 生成模式 | 首次生成、清理演示数据后重建 |
| 生成要求 | 租户可补充文字描述现场要求，作为 AI 命名和演示摘要的提示词输入 |

#### 4.14.6 清理与安全

生成的演示数据必须带以下标记：

- `tenant_id`
- `is_demo`
- `demo_batch_id`
- `demo_generated_by`
- `demo_generated_time`

清理规则：

- 只能清理当前租户的演示数据。
- 只能清理带 `is_demo = true` 或指定 `demo_batch_id` 的数据。
- 不能删除客户手工录入的正式数据。
- 清理前必须展示影响范围和确认提示。

### 4.16 核心数据对象

以下为产品层对象规划，不等同于最终数据库表结构。研发落地时可结合框架生成器拆分主表、明细表、流水表、锁定表和附件表。

| 对象 | 核心字段 | 主要关系 |
|---|---|---|
| 仓库 | 仓库编码、名称、类型、负责人、状态 | 下挂库区、库位 |
| 库区 | 库区编码、名称、仓库、用途、状态 | 下挂库位 |
| 库位 | 库位编码、库区、容量、适用品类、是否混放、状态 | 承载库存批次 |
| 物料档案 | 物料编码、名称、规格、单位、分类、批次管理标识 | 关联库存、条码、出入库明细 |
| 供应商 | 供应商编码、名称、联系人、状态 | 关联到货和批次来源 |
| 客户 | 客户编码、名称、联系人、状态 | 关联销售出库 |
| 条码规则 | 规则编码、对象类型、编码段、启用状态 | 生成物料、批次、库位、单据条码 |
| 到货单 | 到货单号、供应商、物料、到货数量、状态 | 可生成收货、检验、入库 |
| 检验任务 | 任务编号、到货单、物料批次、检验结果、状态 | 影响库存状态 |
| 入库单 | 入库单号、入库类型、来源单据、状态 | 生成库存批次和库存流水 |
| 出库单 | 出库单号、出库类型、来源单据、状态 | 消耗库存批次并形成流水 |
| 生产领料单 | 工单、BOM需求、拣货批次、领料状态 | 与 MES 工单联动 |
| 库存批次 | 物料、批次号、仓库、库位、库存状态、数量 | 实时库存查询和追溯核心 |
| 库存余额 | 物料、仓库、库位、批次、可用/锁定/冻结/待检数量 | 由库存流水计算或汇总 |
| 库存流水 | 业务类型、来源单据、物料、批次、数量、前后库存 | 所有库存变更必须记录 |
| 库存锁定 | 来源单据、物料、批次、锁定数量、释放状态 | 支撑出库占用和领料占用 |
| 移库调拨单 | 原仓库/库位、目标仓库/库位、物料、数量、状态 | 生成移库流水 |
| 盘点单 | 盘点范围、盘点方式、差异数量、审核状态 | 审核后生成库存调整 |
| 库存预警 | 预警类型、物料、仓库、等级、处理状态 | 进入通知/待办中心 |

- `tenant_id`

- `updated_id`
- `created_time`
- `updated_time`
- `is_demo`
- `demo_batch_id`

### 4.17 关键状态流转

| 对象 | 状态流转 |
|---|---|
| 到货单 | 待收货 -> 已收货 -> 待检验 -> 待入库 -> 已入库/已退货 -> 已关闭 |
| 检验任务 | 待检验 -> 检验中 -> 待判定 -> 合格/不合格/让步接收 -> 已关闭 |
| 入库单 | 待上架 -> 上架中 -> 已入库 -> 已取消 |
| 出库单 | 待拣货 -> 拣货中 -> 待复核 -> 已出库 -> 已取消 |
| 生产领料单 | 待备料 -> 待拣货 -> 待确认 -> 已领料 -> 已关闭 |
| 库存批次 | 待检 -> 可用 -> 锁定 -> 已出库；可用 -> 冻结 -> 解冻/不良 |
| 移库调拨单 | 待执行 -> 执行中 -> 已完成 -> 已取消 |
| 盘点单 | 草稿 -> 已发布 -> 盘点中 -> 待审核 -> 已调整 -> 已关闭 |
| 库存预警 | 已触发 -> 待处理 -> 处理中 -> 已处理 -> 已关闭 |

状态规则：

1. 待检库存不可直接发料或销售出库。
2. 冻结库存不可被推荐出库批次。
3. 已锁定库存不可被其他单据重复占用。
4. 盘点审核前不影响正式库存，审核后才生成调整流水。
5. 所有库存数量变化必须先写库存流水，再更新库存余额。

### 4.18 端到端业务流程

#### 4.18.1 采购到货入库闭环

```text
采购到货创建/同步
  -> 收货登记
  -> 生成来料检验任务
  -> 检验判定
  -> 合格入库/让步入库/退货/冻结
  -> 推荐库位
  -> 扫码上架
  -> 生成库存批次和库存流水
  -> 批次追溯归档
```

#### 4.18.2 生产领料闭环

```text
MES 工单/BOM 需求
  -> WMS 齐套检查
  -> 生成备料/领料任务
  -> 推荐出库批次
  -> 库存锁定
  -> 扫码拣货
  -> 生产领料员确认
  -> 库存扣减
  -> 批次回传 MES
```

#### 4.18.3 成品入库出库闭环

```text
MES 工单完工/出厂试验合格
  -> 生成成品入库任务
  -> 扫码入库
  -> 形成成品批次库存
  -> 销售出库任务
  -> 推荐出库批次
  -> 拣货复核
  -> 出库确认
  -> 客户批次追溯
```

#### 4.18.4 盘点调整闭环

```text
创建盘点计划
  -> 发布盘点单
  -> 扫码盘点
  -> 生成盘盈/盘亏差异
  -> 复核差异原因
  -> 审核
  -> 生成库存调整流水
  -> 更新库存余额
```

### 4.19 外部接口边界

WMS 与外部系统通过 API 或消息方式对接，不通过共享数据库实现业务联动。

| 对接方 | WMS 输入 | WMS 输出 | V1.0 建议 |
|---|---|---|---|
| ERP | 采购订单、销售订单、物料、供应商、客户 | 入库结果、出库结果、库存余额 | 先支持 REST/导入，后续做自动同步 |
| MES | 工单、BOM需求、完工入库需求、齐套请求 | 可用库存、缺料结果、领料批次、成品入库结果 | 两系统独立库，通过 API 对接 |
| PDA/H5 | 扫码请求、作业确认 | 作业任务、校验结果、库存状态 | V1.0 可先 H5 移动适配 |
| RFID/电子秤 | 识别结果、重量数据 | 业务校验结果 | V1.0 预留接口，不强依赖硬件 |
| 平台 AI | 平台默认模型能力 | 库存摘要、差异说明、预警建议、试用数据文案 | WMS 不维护模型密钥 |
| 文件服务 | 附件上传、模板存储 | 到货附件、检验附件、盘点附件、条码模板 | 复用框架上传能力 |

### 4.20 验收标准

| 范围 | 验收口径 |
|---|---|
| 仓库库位 | 可维护仓库、库区、库位，支持启停和查询 |
| 条码规则 | 可生成物料、批次、库位、单据条码 |
| 到货检验 | 到货可收货、生成检验、按检验结果控制入库 |
| 入库管理 | 可扫码上架，入库后库存批次和库存流水正确 |
| 出库管理 | 可推荐批次、扫码拣货、复核出库，库存扣减正确 |
| 生产领料 | 可接收 MES 工单需求，生成领料任务并回传批次 |
| 实时库存 | 可按物料、仓库、库位、批次查询可用/锁定/冻结/待检数量 |
| 库存盘点 | 可创建盘点、录入差异、审核后生成调整流水 |
| 库存预警 | 可自动生成安全库存、缺料、呆滞预警 |
| 批次追溯 | 输入物料批次或成品批次可查完整出入库流向 |
| 智能能力 | 可自动推荐库位/批次，驾驶舱可显示智能摘要 |
| 试用初始化 | 新租户可生成、清理并重建演示数据，不影响正式数据 |
| 权限数据 | 不同角色只能看到授权菜单、按钮和仓库数据范围 |
| 审计日志 | 入库、出库、移库、冻结、盘点调整等关键动作有日志 |

## 6. 待确认事项
