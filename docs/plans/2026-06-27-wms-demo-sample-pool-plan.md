# WMS 演示数据样本池实施规划

## 1. 目标

建设一套“电工装备样本池”，专门服务 WMS 租户演示数据生成。样本池用于提示词、规则模板和默认行业语料，不作为正式业务种子、不创建全局物资目录，也不影响租户正式物料分类。

目标效果：

- 租户管理员可基于默认样本池快速生成丰富演示数据。
- 租户可复制默认样本池并调整提示词、产品方向、命名风格和数据偏好。
- 生成器按租户画像、样本池和数据规模生成仓库、库位、物料、供应商、库存、单据、预警、追溯和驾驶舱数据。
- AI 不可用时仍使用样本池内置模板生成可用数据。

## 2. 产品边界

样本池是“演示数据生成配置”，不是正式主数据。

| 项目 | 决策 |
|---|---|
| 附件物资范围 | 只作为默认样本池语料来源 |
| 是否 seed 到正式表 | 否 |
| 是否新增物资目录业务模块 | 否 |
| 是否影响正式物料分类 | 否 |
| 是否租户可改 | 是，租户只能改自己的样本池副本 |
| 是否支持 AI | 是，但规则生成优先，AI 只增强文案和命名 |

## 3. 样本池结构

建议新增 WMS 专属配置表，不放在正式仓储主数据表里。

### 3.1 `wms_demo_sample_pool`

用于记录样本池主信息。

| 字段 | 说明 |
|---|---|
| `tenant_id` | 租户 ID；默认系统池可使用平台租户或 `0` |
| `code` | 样本池编码，如 `electrical_equipment_default` |
| `name` | 样本池名称，如“电工装备默认样本池” |
| `industry` | 行业，如 `electrical_equipment` |
| `is_system` | 是否系统内置 |
| `is_active` | 是否启用 |
| `base_pool_id` | 租户副本来源 |
| `prompt_template` | 生成提示词模板 |
| `fallback_template` | AI 不可用时的模板 |
| `config` | JSON 配置，保存数据规模、词库权重、默认场景 |

### 3.2 `wms_demo_sample_item`

用于存放可编辑样本项。

| 字段 | 说明 |
|---|---|
| `pool_id` | 样本池 ID |
| `group_key` | 样本组，如 `coil`、`switch`、`cable` |
| `group_name` | 样本组名称，如线圈、开关、线缆 |
| `item_key` | 样本项编码 |
| `item_name` | 样本项名称，如变压器、高压电缆 |
| `acceptance_scope` | 受理范围/能力描述 |
| `spec_patterns` | 规格型号模板数组 |
| `supplier_patterns` | 供应商命名模板数组 |
| `material_patterns` | 物料命名模板数组 |
| `storage_traits` | 仓储特性，如重货、盘具、易损、需检 |
| `quality_traits` | 检验特性 |
| `enabled` | 是否启用 |

## 5. 租户可编辑能力

租户管理员在“试用数据 / 样本池配置”中管理自己的样本池。

第一版能力：

1. 查看系统内置样本池。
2. 一键复制为租户样本池。
3. 编辑样本池名称、启用状态、提示词模板。
4. 编辑样本组和样本项的启用状态、权重、命名模板、规格模板。
5. 试用数据初始化时选择样本池。

限制：

- 租户不能修改系统内置池。
- 租户样本池只对本租户可见。
- 删除样本池不删除已经生成的演示数据。
- 正式物料和供应商不反向引用样本池。

## 6. 生成流程

```text
租户填写企业画像
  -> 选择样本池、产品方向、数据规模、时间范围
  -> 规则生成器读取样本池
  -> 生成结构化计划
  -> 可选调用平台 AI 优化名称、规格、摘要和异常说明
  -> 按计划落库演示数据
  -> BusinessTask 记录进度、结果、错误和 counts
```

规则生成器负责：

- 主外键、数量、库存状态、时间线、单据状态流。
- 仓库、库区、库位数量。
- 物料、供应商、客户、批次、库存余额、库存流水。
- 到货、检验、入库、出库、领料、调拨、盘点、冻结、预警、追溯。

AI 负责：

- 仓库/库区/库位自然命名。
- 物料名称、规格型号、供应商名称。
- 来料异常、盘点差异、安全库存预警说明。
- 生成结果摘要和租户试用场景说明。

## 7. 数据规模与用户调节

第一版支持固定模式和自定义模式。固定模式用于降低普通租户理解成本，自定义模式用于管理员按演示目标调节数量。后端必须二次校验并裁剪范围，不能只依赖前端限制。

| 模式 | 仓库 | 库位 | 物料 | 库存流水 | 业务单据 | 适用场景 |
|---|---:|---:|---:|---:|---:|---|
| 快速体验 | 1-2 | 20-60 | 20-50 | 50-150 | 20-60 | 快速看效果、销售演示 |
| 标准演示 | 3-5 | 100-300 | 80-200 | 300-800 | 120-300 | 默认客户试用 |
| 丰富演示 | 5-8 | 300-800 | 200-500 | 1000-3000 | 400-1000 | 深度体验 |
| 自定义 | 1-8 | 10-1000 | 10-600 | 20-5000 | 20-1500 | 高级管理员按需配置 |

普通租户默认开放快速体验和标准演示。丰富演示可受套餐或配置限制。

用户自定义数量时，界面应同时显示预计生成耗时、预计影响行数和清理范围。生成前先调用预览接口，确认后再创建任务。

## 8. API 规划

| API | 方法 | 权限 | 说明 |
|---|---|---|---|
| `/wms/demo/sample-pools` | `GET` | `module_wms:demo:init` | 查询可用样本池 |
| `/wms/demo/sample-pools/{id}` | `GET` | `module_wms:demo:init` | 查看样本池详情 |
| `/wms/demo/sample-pools/{id}/copy` | `POST` | `module_wms:demo:config` | 复制系统池为租户池 |
| `/wms/demo/sample-pools/{id}` | `PUT` | `module_wms:demo:config` | 编辑租户池 |
| `/wms/demo/sample-items/{id}` | `PUT` | `module_wms:demo:config` | 编辑租户样本项 |
| `/wms/demo/init` | `POST` | `module_wms:demo:init` | 增加 `sample_pool_id`、`scale_mode`、`time_range_days` |

## 9. 前端规划

菜单仍放在 WMS 试用数据能力下，不单独做正式产品目录。

页面：

1. **试用数据初始化**
   - 企业名称、行业、企业规模、产品方向、数据规模、时间范围、样本池选择。
   - 支持用户填写自有产品：产品名称、品类、规格示例、电压等级、仓储特性、质量要求、供应商要求。
   - 支持“固定框 + 补充说明”：固定框负责结构化数量和产品，补充说明负责表达客户现场要求。
   - 生成前展示预计生成数量、产品混合、工作流覆盖和影响范围。

2. **样本池配置**
   - 系统内置池只读。
   - 租户池可编辑。
   - 支持样本项启停、权重、模板编辑、提示词编辑。
   - 租户修改的是自己的副本，不影响系统池和其他租户。

3. **生成历史**
   - 展示 `BusinessTask`、`demo_batch_id`、counts、耗时、错误、清理入口。

4. **预览与质量报告**
   - 预览接口只返回计划，不落库。
   - 质量报告展示是否覆盖到货、检验、入库、出库、领料、调拨、盘点、预警、追溯。
   - 如果标准演示未达到最低数量或流程覆盖，禁止提交生成任务。

## 10. 实施步骤

1. 增加样本池规划到 PRD，并明确非正式 seed 边界。
2. 新增样本池模型、迁移、schema、service、controller。
3. 内置电工装备默认样本池，以代码 fixture 初始化系统池。
4. 改造 `WmsDemoGenerator`：从固定数据改为“样本池 + 规则计划”生成。
5. 扩展 demo init schema：`sample_pool_id`、`product_directions`、`scale_mode`、`time_range_days`。
6. 增加前端样本池选择和配置页面。
7. 增加测试：租户副本隔离、系统池不可改、生成数量达标、清理不删正式数据。
8. 实测：快速体验、标准演示、AI 可用、AI 不可用四种路径。

## 11. 验收标准

- 新租户无需人工维护正式物料，也能生成行业特征明显的演示数据。
- 租户可修改自己的样本池提示词和样本项，不影响其他租户和系统内置池。
- 标准演示至少生成 3 个仓库、80 个物料、300 条库存流水，并覆盖到货、检验、入库、出库、领料、调拨、盘点、预警、追溯。
- 所有演示数据都有 `tenant_id`、`is_demo`、`demo_batch_id`。
- 清理只删除当前租户当前批次演示数据。
- AI 不可用时生成流程仍成功。
